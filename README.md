# Bienvenue dans votre test technique Android !

Vous disposez de 7 jours pour effectuer ce test technique et nous renvoyer le lien vers votre repository

## Voici que l'on attend de vous dans ce test :
 
- Cloner le repository et pousser vos modifications sur un repository de votre compte github
- Utiliser la clean archi
- Utiliser des flow
- Ajouter Ktlint et Mockito
- Modéliser et mettre en place une base de données en s’inspirant du fichier json
- Implémenter les 3 User Stories qui vont suivre

### User Story 1 : 
En tant qu’Auditeur, je veux pouvoir consulter la liste de mes équipements. Je veux pouvoir faire une recherche dans mes équipements sur leur nom.
Je veux pouvoir sélectionner un équipement pour voir plus d’informations sur cet équipement

### User Story 2 : 
En tant qu’Auditeur,  je peux pouvoir consulter les informations d’un équipement donné, et je veux pouvoir modifier :

- La marque → Doit commencer par une majuscule et ne pas dépasser 25 caractères.
- Le modèle → Ne doit pas dépasser 50 caractères.
- Le numéro de série → Contient uniquement des chiffres, des lettres et des tirets. Tous les caractères doivent être en majuscule. Ne peut pas contenir d’espaces. Ne doit pas dépasser 50 caractères

### User Story 3 ( Bonus ) : 
En tant qu’Auditeur, je veux pouvoir ajouter des équipements depuis la liste.
